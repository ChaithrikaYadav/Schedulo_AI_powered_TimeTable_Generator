# Schedulo — Model Artefacts Directory

This directory stores **trained ML model files** produced by Schedulo's ML pipeline (Phase 3).

---

## Model Files (Generated at Runtime)

| File | Algorithm | Phase | Purpose |
|------|-----------|-------|---------|
| `quality_predictor.pkl` | XGBoost `XGBRegressor` | Stage 4 | Predicts timetable quality score (0–1) from scheduling features |
| `anomaly_detector.pkl` | `IsolationForest` (sklearn) | Stage 6 | Detects abnormal timetable patterns (e.g. overloaded days) |
| `rl_policy.zip` | PPO (Stable-Baselines3) | Stage 5 | RL fine-tuning policy for constraint satisfaction improvement |

> **Note:** These files are NOT committed to git (see `.gitignore`). They are generated by running the training scripts below.

---

## How to Train Models

### Quality Predictor (XGBoost)
```bash
python models/train_quality_predictor.py
# Or with synthetic data (no DB needed):
python models/train_quality_predictor.py --dry-run
```

### Anomaly Detector (Isolation Forest)
```bash
python models/train_anomaly_detector.py
# Or with synthetic data:
python models/train_anomaly_detector.py --dry-run
```

---

## Model Registry

Use `ModelRegistry` to load models safely at inference time:

```python
from models.model_registry import ModelRegistry

registry = ModelRegistry()
model = registry.load("quality_predictor")  # Returns None if not trained yet
if model:
    score = model.predict([[features...]])[0]
```

## Feature Schema (Quality Predictor)

| Feature | Description |
|---------|-------------|
| `conflict_count` | Number of scheduling conflicts detected |
| `generation_time_ms` | How long the scheduler ran (normalised) |
| `slot_fill_ratio` | Fraction of slots that have a class assigned |
| `lab_pair_ratio` | Fraction of lab slots that are properly paired |
| `free_period_std` | Standard deviation of free slots per day (consistency) |
| `faculty_util_ratio` | Average faculty utilisation vs weekly limit |
